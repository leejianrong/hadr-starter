The rules of the desk

Why you can trust the numbers.

Four commitments the pipeline enforces in code — not in good intentions.

Severity is impact, not magnitude.

We rank on the colour PAGER and GDACS already computed from population exposure and vulnerability. We never estimate casualties ourselves.

Figures are attributed, never summed.

A modelled estimate and a government count are two readings of one reality — shown side by side with their sources, never added into a false total.

Quiet is information.

The page refreshes every morning even when nothing crossed the line, so silence means calm — not a broken pipe.

Deterministic by design.

Every decision — what shows, how it ranks, what merged — is code you can read and test. A language model only ever writes the prose, never a number.

Three feeds · three kinds of truth

Not three views of one thing.

Each feed is a different object type at a different speed. The desk reads all three and resolves them into one situation per line.

USGS Earthquakes, in seconds.

A physical event, machine-measured — the ground-truth reading for where and how hard the earth moved.

seconds-latent · earthquakes only
GDACS Impact alerts, in minutes.

Earthquakes, cyclones, floods, wildfires and volcanoes — each colour-coded by modelled human impact, versioned as the event evolves.

minutes-latent · EQ / TC / FL / WF / VO
ReliefWeb Curated crises, in days.

Everything above, plus epidemics and conflict. If humanitarian editors made a page for it, it has already cleared the bar for attention.

days-latent · + epidemic + conflict

The same earthquake can arrive from all three feeds. Because a GDACS quake is built from the USGS reading, agreement between them isn't corroboration — it's one source counted twice. The desk knows the difference, and merges them into a single line.

How it works

Deterministic first. The model comes last, and only sometimes.

fetch› normalize› decluster› cross-feed join› threshold› render› narrate · loud only

Every morning a scheduled job fetches the three feeds, normalizes them to a common model, declusters aftershock swarms, resolves the same event across feeds, applies a fixed attention threshold, and renders the page — all in plain, tested Python. Only if something material changed since yesterday is a language model woken to write the short narrative at the top. It never decides what's shown, and never touches a figure. The result is committed and published — the same page, byte-for-byte, on a quiet morning.

Coverage & limits

What this doesn't claim.

  • No casualty modelling. We consume the figures the experts publish; we don't forecast exposure or invent counts.
  • "Onshore" is approximated by country outline, so a quake just off a coast can read as offshore. The page says so.
  • ReliefWeb runs in RSS mode for now — status and full backfill are limited until the API is enabled. Flagged on every report.
  • Coverage is bounded: USGS earthquakes, GDACS hazards, and ReliefWeb-curated crises — not everything, and the banner never pretends otherwise.
