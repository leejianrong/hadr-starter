What you get Trust Feeds Report

The world's disasters, ranked by human impact.

Twice a day, at 08:30 and 15:00 Singapore time, HADR Monitor reads three global feeds and shows the disasters that need attention, with the highest human impact first.

One page, two sections.

Fast events and slow crises run on different clocks, so the report keeps them apart.

Sudden onset, last 24 hours

Sudden events

Earthquakes, cyclones, floods, and wildfires from the last 24 hours, with the past 7 days kept below for context. Aftershocks are grouped under their main quake. An event reported by two feeds shows once, not twice.

Slow onset, always shown

Ongoing crises

Droughts, epidemics, and conflict have no single start time. They stay on the page for as long as they are active, with no 24-hour cutoff.

Why you can trust the numbers.

Four rules, enforced in code.

Severity means impact, not size.

We use the alert level that USGS and GDACS already calculate from population and vulnerability. We do not estimate casualties ourselves.

Sources stay separate.

A modelled estimate and a government count are two views of one event. We show both, each with its source. We never add them together.

Quiet mornings still publish.

The page updates every day, even when nothing is wrong. Silence means calm, not a broken system.

Code decides, not a model.

What shows, how it ranks, and what merges are all fixed in code you can read and test. A language model only writes the summary text, never a number.

Three feeds, three kinds of record.

Each feed is a different kind of record, arriving at a different speed. The report combines them into one line per event.

USGS Earthquakes, in seconds.

The physical measurement of where an earthquake struck and how strong it was.

seconds · earthquakes only
GDACS Impact alerts, in minutes.

Earthquakes, cyclones, floods, wildfires, and volcanoes, each rated by likely human impact.

minutes · EQ, TC, FL, WF, VO
ReliefWeb Curated crises, in days.

Everything above, plus epidemics and conflict. If humanitarian editors made a page for it, it already matters.

days · adds epidemics, conflict

The same earthquake can arrive from all three feeds. A GDACS quake is built from the USGS reading, so the two agreeing is not extra proof. The report knows this and shows them as one line.

How it works.

fetch› clean› group› match across feeds› rank› build page› summarize, only if changed

Each morning a scheduled job fetches the three feeds, cleans them into one format, groups aftershocks, matches the same event across feeds, ranks by impact, and builds the page. All of this is plain, tested Python. Only if something important changed since yesterday does a language model write the short summary at the top. It never chooses what to show, and never changes a number.

What it does not do.

  • It does not estimate casualties. It reports the figures that experts publish.
  • Onshore is approximate. It uses country outlines, so a quake just off the coast can look offshore.
  • ReliefWeb uses the public RSS feed for now, so status and older records are limited. Each report says so.
  • Coverage is limited to USGS earthquakes, GDACS hazards, and ReliefWeb crises. The report states what it does not cover.
